## Aim

Persist scan state, scanner runs, findings, governor decisions, and report metadata using Prisma 7's adapter-based client. Provide repositories that scope every query by `scanId`, never call `findUnique({ where: { id } })` on client-supplied IDs, and wrap multi-row writes in `$transaction`. Ship a regression service that classifies findings as new / persisted / fixed.

### Step 5: ScanRepository (SM-35)

- **File**: `src/persistence/scan.repository.ts` (NEW FILE)
- **Detail**: `create`, `findById`, `updateStatus`, `findRecentByRepo`, `findAllRecent`. Status enum literal type `ScanStatus`. `updateStatus` includes optional `completedAt` and `blueprintMd` only when supplied.

### Step 6: FindingRepository (SM-35)

- **File**: `src/persistence/finding.repository.ts` (NEW FILE)
- **Detail**: `insertMany(scanId, findings)` wraps `createMany` in `$transaction`. `findByFingerprint(scanId, fingerprint)` uses the composite `scanId_fingerprint` key (NEVER `findUnique({ id })` on client input). `findAllByScanId` and `countByScanId` scope by `scanId`.
- **Constraint**: Anti-BOLA: every read scoped by `scanId`. `insertMany` returns 0 for an empty list without touching prisma.

### Step 7: PhaseRunRepository (SM-35)

- **File**: `src/persistence/phase-run.repository.ts` (NEW FILE)
- **Detail**: `create` (status RUNNING by default), `complete` (truncates `rawOutput` to 5 MB cap), `findByScanId`. PhaseRunStatus literal union: `PENDING / RUNNING / COMPLETED / FAILED / TIMED_OUT / SKIPPED`.
- **Constraint**: rawOutput truncation enforced in the parser to prevent unbounded DB rows (Nuclei gotcha from AGF::NucleiScanner).

### Step 8: GovernorDecisionRepository (SM-35)

- **File**: `src/persistence/governor-decision.repository.ts` (NEW FILE)
- **Detail**: `record` (immutable insert), `findByScanId`, `findByScanIdAndPhase`. JSON stringification handled here so callers pass typed objects.
- **Constraint**: NEVER expose an `update` method — decisions are append-only.

### Step 9: RegressionService (SM-36)

- **File**: `src/persistence/regression.service.ts` (NEW FILE)
- **Detail**: `diff(currentScanId)` returns `{ newFingerprints, fixedFingerprints, persistedFingerprints, baselineScanId }`. Baseline = most recent COMPLETED/PARTIAL scan for the same `targetRepo`, started before the current scan. Pure operation: does not write to the DB.
- **Constraint**: Exclude the current scan from the baseline lookup explicitly. Empty / null baseline → all current findings classified as new.

### Step 10: PersistenceModule + barrel + tests + quality gate

- **File**: `src/persistence/persistence.module.ts`, `src/persistence/index.ts`, three `.spec.ts` test files
- **Detail**: NestJS module providing PrismaClient via factory, plus all repositories. Tests mock PrismaClient with `vi.fn()` and verify call arguments, scoping, transaction wrapping, and edge cases (empty list, missing baseline, etc.).
- **Constraint**: Tests must NOT require a real Prisma client or SQLite file.

# Important Findings

- [Step 1] **Prisma 7 breaking change — `url` removed from datasource block**: First attempt with `url = env("DATABASE_URL")` failed with `P1012`. Prisma 7 separates schema parsing from datasource resolution; the URL is now provided via `prisma.config.ts`, the `--url` CLI flag, or the runtime adapter. Schema only declares `provider = "sqlite"`.
- [Step 2] **`prisma.config.ts` is REQUIRED in Prisma 7** for `prisma migrate` and `prisma generate` to know where to find the schema. Minimal config: `{ schema: 'prisma/schema.prisma', migrations: { path: 'prisma/migrations' } }`.
- [Step 3] **Migration applied via `--url` flag**: `pnpm exec prisma migrate dev --name init --url "file:./data/sentinel.db"` worked after the schema config was Prisma-7-compliant. Generated `prisma/migrations/20260411163536_init/migration.sql` with all 5 tables, FKs, indexes, and the unique constraint.
- [Step 4] **Adapter class is `PrismaBetterSqlite3` (lowercase 'qlite')**, NOT `PrismaBetterSQLite3`. TypeScript caught this on the first typecheck — corrected via search-and-replace.
- [Step 4] **better-sqlite3 native binding requires postinstall**: pnpm 10 ignores build scripts by default. Added `pnpm.onlyBuiltDependencies` to package.json including `better-sqlite3`, `prisma`, `@prisma/client`, `@prisma/engines`, `@nestjs/core`, `esbuild`, `msgpackr-extract`. Fresh checkouts must run `pnpm install` (which now triggers the build) and / or `pnpm approve-builds` once.
- [Step 10] **Tests mock PrismaClient at the surface used by each repository** — no Prisma test extensions, no test database, no `vitest-mock-extended`. Each repository's test creates a minimal `prisma` proxy with `vi.fn()` for the methods the repository calls. The `$transaction` mock invokes its callback synchronously with the same proxy as the `tx`.
- [Step 10] **`prisma.config.ts` excluded from ESLint**: ESLint's `projectService` requires every linted file to be in a tsconfig include path. `prisma.config.ts` is a tooling config at the repo root, not application code, so it's added to the eslint ignores list rather than to `tsconfig.json`.
- [Steps 5–9] **Repository constructor injection**: Every repository takes `PrismaClient` via constructor — they do NOT call `createPrismaClient()` themselves. This makes tests trivially mockable and lets the PersistenceModule swap the client at the DI level.
